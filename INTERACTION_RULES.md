# 智能单词学习网站交互与业务规则

版本：v0.1  
日期：2026-07-03  
用途：补充 PRD 中不适合展开过细的页面状态、按钮逻辑、计算规则和边界情况。

## 1. 基本原则

- 学习按“次”推进：用户每次进入词库学习，都会形成一个学习会话。
- 复习按“天”管理：系统根据单词到期时间生成每日复习目标。
- 首页是学习中枢：学习和复习合并展示在首页，不再使用独立顶部 Tab。
- PRD 负责产品目标、页面和核心流程；本文件负责细规则和边界情况。

## 2. 首页规则

### 2.1 页面布局

- 首页内容区居中展示。
- 页面顶部左侧展示产品名称。
- 页面顶部右侧展示主色按钮“设置 icon”。
- 首页主体由上下两张大卡片组成：
  - 学习
  - 复习

### 2.2 学习大卡片

学习大卡片展示所有词库。

词库卡片展示：

- 词库名称
- 已学 / 总数
- 学习进度条
- 本次已学数量
- 开始学习 / 继续学习按钮
- 更多 icon

本次已学数量展示规则：

- 如果当前词库存在未提交测验的学习会话，且已经标记过至少 1 个词，则展示“本次已学 x 个”。
- 如果当前词库没有进行中的学习会话，不展示该行。
- 不展示“今日未学习”。
- 不展示“今日已完成”。

按钮文案：

- 首页词库卡片统一展示“立即学习”。
- 用户每次点击“立即学习”都创建一个新的学习会话，不复用上一次未完成会话。

点击立即学习：

- 创建新的学习会话。
- 默认加入 20 个待学习词。
- 进入学习会话页。

### 2.3 词库更多 icon

词库卡片右上角展示更多 icon。

点击更多 icon：

- 打开右侧词库详情 Drawer。
- 不在卡片上直接展示删除按钮。
- Drawer 内后续可扩展词库详情、词表、导出、删除等功能。

### 2.4 复习大卡片

复习大卡片重点展示今日复习进度。

展示字段：

- 今日复习进度
- 待复习数量
- 已过期数量
- 掌握率
- 连续复习天数
- 立即复习按钮

不展示薄弱词列表。

今日复习进度计算：

```txt
今日复习目标 = min(今日到期词数量, 每日复习上限)
今日复习进度 = 今日已复习数量 / 今日复习目标
```

如果今日复习目标为 0：

- 进度显示为“今日暂无待复习”。
- 立即复习按钮可置灰，或进入空状态页。

## 3. 设置弹窗规则

点击首页右上角“设置 icon”按钮后打开设置弹窗。

设置项：

- DeepSeek API Key
- 默认每次学习数量，默认 20
- 每日复习上限，默认 30
- 数据管理暂不实现

DeepSeek API Key 保存在本机浏览器，用于 AI 段落翻译测评。未配置 API Key 时，提交测评应提示用户先进入设置配置。

学习目标保存后：

- 后续新建学习会话使用新的“每日单词学习数量”。
- 后续进入复习页使用新的“每日复习目标”。

## 4. 学习会话规则

### 4.1 创建学习会话

用户点击词库卡片“立即学习”时创建学习会话。

创建规则：

- 每次点击都创建新的学习会话。
- 默认取 20 个待学习词。
- 优先选择该词库中未学过的词。
- 如果未学过词不足 20 个，则补充需要复习或掌握度较低的词。
- 如果词库剩余可学习词不足 20 个，则取全部可用词。

### 4.2 学习会话状态

学习会话中的单词状态：

- pending：待学习
- known：认识
- familiar：有印象
- unknown：不认识

用户点击标记按钮后：

- 单词从左侧待学习区移除。
- 单词进入右侧对应分类。
- 学习会话记录该词状态和标记时间。

### 4.3 撤回标记

用户 hover 右侧标签时显示关闭按钮。

点击关闭按钮：

- 单词从右侧分类中移除。
- 单词回到左侧待学习区。
- 单词状态恢复为 pending。
- 本次已学数量减少。

### 4.4 再学 10 个

用户点击“再学 10 个”：

- 从当前词库继续追加 10 个待学习词。
- 如果可用词不足 10 个，追加全部可用词。
- 不影响右侧已标记结果。
- 不新建学习会话。

### 4.5 进入测验

进入测验按钮规则：

- 用户至少标记 1 个词后可进入测验。
- 如果左侧仍有待学习词，可以进入测验，但页面可提示“还有未标记单词”。
- 如果左侧清空，左侧区域展示完成态，强调“再学 10 个”和“进入测验”。

进入测验后：

- 当前学习会话可以暂存为待测验状态。
- 测验完成后再写入 completedAt。

## 5. Hover 朗读规则

可朗读对象：

- 英文单词
- 英文例句

触发规则：

- 鼠标停留超过 600ms 后触发朗读。
- 鼠标在 600ms 内移出，不触发朗读。
- 新朗读开始时，可中断上一条朗读。
- 同一内容短时间内不重复朗读。
- 用户关闭 hover 自动朗读后，不再触发。

默认朗读设置：

```txt
语言：en-US
语速：0.85
```

## 6. 复习规则

### 6.1 今日复习候选池

候选池包括：

- 已到期单词
- 已过期单词
- 最近测验中需要加强的词

### 6.2 每日复习上限

默认每日复习上限为 30。

```txt
今日复习目标 = min(今日到期词数量, 每日复习上限)
```

如果到期词数量小于等于上限：

- 全部进入今日复习队列。

如果到期词数量大于上限：

- 只取优先级最高的前 30 个。
- 其他词保留过期状态，后续日期继续优先展示。

### 6.3 复习排序

排序公式：

```txt
优先级 = 过期天数 × 2 + (1 - 掌握度) × 3 + 错误次数 × 1
```

排序方向：

- 分数越高，越优先复习。
- 已过期词优先于未过期词。

### 6.4 复习详情页

复习详情页为 Anki 式卡片。

页面顶部可切换词库：

- 默认选中“全部（默认）”。
- 切换词库后重置当前复习队列，并展示所选词库的待复习卡片。
- MVP 阶段使用本地已学习但未掌握的词生成复习队列；后续由后端接口返回对应词库的复习数据。
- 单词只有完成复习流程后才计入“已掌握”。学习页标记“认识”只代表本次学习分拣结果，未完成复习流程前仍计入“复习中”。

正面展示：

- 单词
- 音标
- 点击显示释义

点击卡片后展示背面：

- 中文释义
- 英文例句
- 例句中文翻译
- 展开更多按钮
- 反馈按钮

反馈按钮：

- 瞬间想起
- 需要想想
- 完全忘了

点击反馈后：

- 更新该词下次复习时间。
- 更新掌握度。
- 更新错误次数。
- 进入下一张复习卡片。

## 7. AI 测验规则

### 7.1 选词

从当前学习会话中选取 5-10 个词。

优先级：

```txt
不认识 > 有印象 > 认识
```

### 7.2 结果分级

AI 评判不只输出对错，而是输出三档：

- 准确掌握
- 基本理解
- 需要加强

需要加强的词自动进入复习候选池。
