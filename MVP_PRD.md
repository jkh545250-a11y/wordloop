# 智能单词学习网站 MVP PRD

版本：v0.1  
日期：2026-07-03  
阶段：网页端 MVP，后续适配 H5  

## 1. 产品定位

智能单词学习网站是一个面向个人使用的英语词汇学习工具。产品通过主题词库、批量学习分拣、AI 段落测验和简化间隔重复复习，帮助用户完成从认识单词到语境应用再到复习巩固的学习闭环。

第一阶段重点不是做完整商业化 App，而是做一个可真实使用、可逐步扩展、适合作品集沉淀的 MVP。

## 2. 核心目标

- 支持用户按主题学习单词。
- 支持 AI 生成词库和 AI 生成段落测验。
- 支持学习时快速标记单词状态：认识、有印象、不认识。
- 支持测验结果容错判断，而不是简单对错。
- 支持错词和薄弱词进入复习队列。
- 支持本地数据保存，以及 JSON 导入导出。
- 先完成普通网页端体验，再做响应式 H5 适配。

它不是单张卡片翻页模式，而是批量学习分拣模式。

### 7.7 Hover 朗读

MVP 第一版学习页先预留朗读接口，不默认实现 hover 自动朗读；待学习流程、设置项和浏览器兼容性稳定后再启用。

后续启用时的交互规则：

- hover 单词：朗读该单词。
- hover 英文例句：朗读该英文例句。
- 朗读触发需要延迟，默认 600ms。
- 如果鼠标在 600ms 内移出，不触发朗读。
- 同一内容短时间内不重复朗读，避免频繁打断。
- 用户可以在设置页关闭 hover 朗读。

技术实现：

```txt
使用 Web Speech API 的 SpeechSynthesisUtterance。
默认朗读语言：en-US
默认语速：0.85
```

学习会话页桌面优先。右侧已标记区保持固定或有限宽度，左侧待学习区随视口拉伸。左侧单词卡片需要设置最小宽度和最大宽度，避免窄屏挤压或宽屏过长影响阅读。

## 8. AI 段落测验

### 8.1 进入条件

用户完成或部分完成本次学习会话后，可进入测验。

### 8.2 选词规则

从本次学习会话中选取本轮已标记的全部词生成测验段落。

优先级：

```txt
不认识 > 有印象 > 认识
```

### 8.3 段落生成规则

DeepSeek 生成 80-120 词英文段落，要求：

- 内容自然、连贯。
- 每个目标词至少出现一次。
- 难度适合学习者。
- 只返回英文段落。

### 8.4 翻译输入

用户输入中文翻译。MVP 阶段语音输入为可选功能，优先保证文字输入体验。

## 9. 测验结果页

测验结果页不使用简单对错判断，而是使用容错分级。

### 9.1 结果分级

每个目标词分为三档：

- 准确掌握
- 基本理解
- 需要加强

### 9.2 页面内容

- 总分
- 总体评价
- 逐词分析
- 每个词的用户理解情况
- 建议翻译
- 需要加强的词
- 再来一篇
- 返回首页
- 进入复习

### 9.3 结果处理

- 准确掌握：提高掌握度。
- 基本理解：轻微提高掌握度，保留后续复习。
- 需要加强：加入复习队列，并标记为 hard。

## 10. 复习页

复习页使用以艾宾浩斯遗忘曲线为基础的间隔重复机制。产品文案不需要强调算法复杂度，但系统调度必须围绕“何时最容易遗忘”来安排复习，目标是让用户用尽量少的复习次数完成长期记忆。

### 10.1 复习入口

- 首页复习大卡片进入复习详情页。
- 测验结果页进入复习。
- 首页展示待复习数量。

### 10.2 复习反馈

用户对复习卡片进行反馈：

- 瞬间想起
- 需要想想
- 完全忘了

### 10.3 记忆状态模型

每个已完成测验的单词维护独立记忆状态：

- stage：记忆阶段，越高代表间隔越长。
- ease：难度系数，越高代表该词对用户越容易。
- dueAt：下次复习到期时间。
- lastReviewedAt：最近一次参与学习、测验或复习调度的时间。
- lapses：遗忘次数。
- correctStreak：连续有效回忆次数。
- source：最近一次调度来源，study / quiz / review。
- lastRating：最近一次输入结果。

### 10.4 艾宾浩斯基础间隔

系统使用以下基础间隔作为 stage 对应的下次复习时间：

```txt
stage 0：5 分钟
stage 1：30 分钟
stage 2：12 小时
stage 3：1 天
stage 4：2 天
stage 5：4 天
stage 6：7 天
stage 7：15 天
stage 8：30 天
stage 9：60 天
```

实际间隔 = 基础间隔 × ease。ease 控制在 0.7 - 1.8 之间，避免过度激进或过度保守。

当 stage ≥ 8 且 correctStreak ≥ 8 时，单词计为“已掌握”，不再进入常规复习队列；如果未来复习中被标记为“完全忘记”，则清除掌握状态并重新进入低阶段复习。

### 10.5 三类输入对复习调度的影响

#### 10.5.1 初次学习标记

学习页的三个标记决定初始阶段和难度系数：

```txt
认识：stage 2，ease 1.15
有印象：stage 1，ease 1.00
不认识：stage 0，ease 0.85
```

注意：学习页标记本身不直接计入“已学习”。只有用户完成测验后，本轮标记才正式写入学习记录并生成复习计划。

#### 10.5.2 测验结果

AI 测验结果分为三档：

```txt
答对 / 准确掌握：stage +2，ease +0.08
不完全对 / 基本理解：stage +1，ease -0.02
答错 / 需要加强：stage -1，ease -0.18，correctStreak 归零
```

测验结果会和初次学习标记叠加。例如，用户学习时标记“不认识”，但测验中答对，则不会直接进入长期间隔，而是进入较谨慎的早期阶段；用户学习时标记“认识”，但测验答错，则会被拉回低阶段，优先安排复习。

#### 10.5.3 复习反馈

复习卡片反馈直接决定下一次复习间隔：

```txt
瞬间想起：stage +2，ease +0.10，correctStreak +2
需要想想：stage +1，ease -0.04，correctStreak +1
完全忘记：stage 归 0，ease -0.22，lapses +1，correctStreak 归零
```

为了避免同一轮复习中重复出现刚刚标记过的单词，当前复习会话和当天已复习集合会排除刚复习过的词；但其新的 dueAt 仍会被保存，用于后续复习优先级计算。

### 10.6 复习优先级

```txt
优先级 = 过期小时 × 2 + (掌握目标阶段 - 当前 stage) × 3 + 遗忘次数 × 2 + (1.2 - ease)
```

如果今日到期词数量超过上限，则优先展示：

- 过期更久的词
- stage 更低的词
- lapses 更多的词
- ease 更低的词

### 10.7 每日复习数量

复习按“天”管理，但不是固定每日复习 20 个。系统根据到期词动态生成今日复习目标，并设置每日复习上限。

```txt
每日复习上限：默认 30 个
今日复习目标 = min(今日到期词数量, 每日复习上限)
```

未完成的到期词继续保留为过期状态，后续日期优先展示。

### 10.8 复习详情页

复习详情页使用类似 Anki 的卡片形式。

页面顶部提供词库切换入口：

- 默认展示“全部（默认）”，聚合所有到期复习词。
- 可切换到设计师、程序员、办公、旅游等内置词库，仅复习该词库下的到期词。
- MVP 阶段切换词库会重置当前复习队列，并基于本地已学习、已到期、未掌握的词生成复习数据；后续接入后端后由 API 返回对应词库的复习队列。
- “已掌握”不等同于学习页标记“认识”。单词需要完成复习流程后才计入已掌握；未完成完整复习流程前仍计入复习中。

正面：

- 单词
- 音标
- 点击显示释义

背面：

- 单词
- 音标
- 中文释义
- 英文例句
- 例句中文翻译
- 展开更多按钮
- 瞬间想起 / 需要想想 / 完全忘了

展开更多后显示更多释义、更多例句及中文解释。

## 14. DeepSeek API

### 14.1 生成词库

输入：

- 主题
- 词数
- 难度
- 补充说明

输出 JSON：

```json
[
  {
    "word": "itinerary",
    "phonetic": "/aɪˈtɪnəreri/",
    "meaning": "行程安排",
    "example": "Let me check my itinerary.",
    "exampleTranslation": "让我看一下我的行程安排。",
    "extraMeanings": ["旅行计划", "预定路线"],
    "extraExamples": [
      {
        "sentence": "The itinerary includes three cities.",
        "translation": "这份行程包含三个城市。"
      }
    ],
    "relatedExpressions": ["travel plan", "schedule"]
  }
]
```

### 14.2 生成测验段落

输入：

- 目标词列表
- 难度
- 主题上下文

输出：

- 英文段落

当前实现：

- 用户从学习页进入测验时，前端使用本地保存的 DeepSeek API Key 调用 DeepSeek 生成测验段落。
- 段落生成成功后才进入段落翻译页。
- 切换“英译中 / 中译英”时重新生成对应方向的测验段落。
- 未配置 API Key 时提示用户进入设置配置，不再静默使用 mock 段落。

### 14.3 评判翻译

输入：

- 英文原文
- 用户中文翻译
- 目标词列表

输出 JSON：

```json
{
  "score": 82,
  "overallComment": "整体理解较好，但部分词汇在语境中的含义需要加强。",

    {
      "word": "cuisine",
      "level": "weak",
      "userUnderstanding": "烹饪",
      "suggestedTranslation": "美食，菜肴",
      "comment": "该语境中更接近当地美食，而不是烹饪动作。"
    }
  ]
}
```
